# Stick Art and Asset Planning Guide

## Purpose

This document outlines a practical art/asset plan for Stick. It assumes little or no prior 3D art experience and focuses on getting usable, stylized assets into the game without blocking engineering progress.

Stick should not start with a large custom art pipeline. Start with simple placeholder forms, prove the gameplay and scale, then improve assets in layers.

## Visual Target

### Direction

- Stylized, readable, low-to-mid poly forest survival.
- Naturalistic proportions and scale, but not photorealistic.
- Inspired by Firewatch-like color readability, broad shapes, strong silhouettes, and painterly lighting.
- Realistic enough for navigation: trees, ridges, rocks, clearings, rivers, trails, and slopes must be recognizable from a distance.

### Important Visual Priorities

1. **Landmark readability**
   - Unique terrain silhouettes and large environmental anchors matter more than tiny details.

2. **Performance**
   - Large outdoor world means repeated assets, instancing, LODs, and simple geometry.

3. **Consistent style**
   - A small cohesive asset set is better than many mismatched assets.

4. **Scale believability**
   - Since `1 unit = 1 meter`, asset dimensions should be realistic enough to support navigation and hiking feel.

## Asset Categories

### 1. Terrain Assets

These are mostly generated by code but need materials/textures.

Needed eventually:

- Ground/soil material
- Dry grass material
- Pine needle forest floor material
- Rock/cliff material
- Mud material
- Trail/path material
- Shallow water material
- Snow placeholder if higher elevations are added later

For the first prototype:

- One terrain material with vertex colors or simple splat blending.
- Color zones for grass, dirt, rock, and trail.

### 2. Vegetation

This will be the largest asset category.

Minimum useful set:

- Pine tree small
- Pine tree medium
- Pine tree large
- Dead standing tree
- Fallen log
- Bush/shrub
- Grass clump
- Dry grass clump
- Fern or leafy plant

Later additions:

- Multiple tree species
- Burned tree variants
- Stumps
- Saplings
- Wildflower clumps
- Edible/poisonous plant variants

Important notes:

- Trees should have strong silhouettes.
- Make 3–5 variants per important prop rather than one perfect model.
- Vegetation should support instancing or thin instancing in Babylon.js.

### 3. Rocks and Landmarks

Needed for navigation and world identity.

Minimum useful set:

- Small rock
- Medium boulder
- Large boulder
- Rock outcrop
- Cliff face segment
- Distinct landmark rock formation

Later additions:

- Caves/overhangs
- Scree fields
- River stones
- Dry creek bed stones

### 4. Player Items

Starting inventory:

- Hunting knife
- Lensatic compass
- Trench shovel

For MVP, these can be simple first-person view models or even 2D/flat UI representations.

Recommended order:

1. Compass, because navigation is core.
2. Knife, because it is simple and useful for interaction tests.
3. Shovel, once terrain mutation/digging is real.

### 5. Environmental Props

Minimum useful set:

- Stick/branch
- Small log
- Campfire ring placeholder
- Trail marker or cairn
- Animal track decal/mesh placeholder
- Water source markers: creek edge rocks, reeds, wet mud

Later additions:

- Abandoned camp remnants
- Fire lookout tower
- Ranger signs
- Old fence posts
- Trail blazes
- Backpack/debris/story objects

### 6. UI and 2D Assets

Even with no HUD map, some UI is needed.

Needed:

- Main menu background/style
- Pause menu
- Settings screen
- Save/load indicators
- Debug overlay style
- Inventory item icons
- Compass UI/inspection overlay if not fully 3D

Keep UI minimal and grounded.

## Recommended Tools

### 3D Modeling

**Blender**

- Free and industry-standard enough.
- Can export `.glb`/`.gltf`, which works well with Babylon.js.
- Best long-term choice if learning asset creation.
- Especially good for organic/stylized environment assets: trees, rocks, grass, logs, terrain details, and natural landmarks.

**Fusion 360**

Fusion 360 is acceptable for some Stick assets, especially if you already know it from 3D printing.

Best uses:

- Lensatic compass
- Hunting knife
- Trench shovel
- Cans, tools, containers, brackets, hinges, and mechanical props
- Signs, simple structures, fire lookout/ranger equipment, and other hard-surface objects

Not ideal for:

- Trees
- Rocks
- Bushes
- Grass
- Fallen logs
- Organic natural landmarks

Main concerns:

- CAD geometry can be too dense for games.
- Exports can be overly precise, triangulated oddly, or visually too clean.
- Materials/UVs may need cleanup before runtime use.

Recommended Fusion workflow:

```text
Fusion 360 model
→ export OBJ/FBX/STL
→ import into Blender
→ simplify/clean geometry if needed
→ set origin, scale, and materials
→ export GLB
→ load in Babylon
```

Practical recommendation:

- Use Fusion 360 for hard-surface hero items like the compass, knife, and shovel.
- Use Blender for organic assets and final game-export cleanup.

### Texture/Material Creation

Beginner-friendly options:

- Blender procedural materials
- Krita for hand-painted textures
- GIMP if already familiar
- Material Maker for procedural textures
- Poly Haven for free reference/HDRIs/textures, if license-compatible

### Image Editing / Concepting

- Krita
- GIMP
- Photoshop/Affinity if already owned

### Asset Format

Recommended runtime format:

- `.glb` for 3D models
- `.png` or `.webp` for textures
- `.ktx2` later for compressed GPU textures if needed

## Suggested Art Pipeline

For Blender-created assets:

```text
Concept/reference
→ Simple blockout in Blender
→ Assign basic materials/colors
→ Export GLB
→ Import into Babylon test scene
→ Check scale/performance/silhouette
→ Iterate
→ Add variants/LODs later
```

For Fusion-created hard-surface assets:

```text
Reference images/measurements
→ Model in Fusion 360
→ Export OBJ/FBX/STL
→ Import into Blender
→ Simplify/cleanup if needed
→ Assign game materials
→ Set origin and scale
→ Export GLB
→ Import into Babylon test scene
```

## Babylon.js Asset Rules

### Scale

Use meters.

Examples:

- Small pine: 4–7m tall
- Medium pine: 8–15m tall
- Large pine: 16–30m tall
- Person eye height: roughly 1.6–1.8m
- Door-sized object reference: about 2m tall

### Origin/Pivot

For most world props:

- Put origin at ground contact point.
- Z or Y orientation depends on Blender/Babylon export conventions, but test and standardize early.
- Tree origin should be at trunk base center.
- Rock origin should be at bottom center or natural placement center.

### Naming

Use clear names:

```text
pine_small_a.glb
pine_small_b.glb
pine_medium_a.glb
rock_boulder_large_a.glb
item_compass_lensatic_a.glb
```

Inside Blender, name meshes/materials cleanly too:

```text
mesh_pine_small_a_trunk
mesh_pine_small_a_needles
mat_bark_warm
mat_pine_needles_dark
```

### Materials

Start simple:

- One or two materials per prop.
- Avoid complex node trees at first.
- Use flat colors or simple hand-painted gradients.

For low-poly style, vertex colors can be very useful.

### Performance Guidelines

Early rough targets:

- Grass clump: very low poly, preferably instanced
- Small rocks: under 100–300 triangles
- Medium rocks: under 500–1,500 triangles
- Trees: under 1,000–5,000 triangles depending on size and usage
- Hero items: can be higher, maybe 5,000–20,000 triangles

These are not strict rules. Test in the actual game.

## Placeholder Strategy

Do not wait for finished art.

Use three levels of assets:

### Level 0: Debug Geometry

- Cubes
- Cylinders
- Cones
- Colored materials
- Procedural simple meshes

Purpose: prove systems.

### Level 1: Blockout Art

- Simple Blender-made props
- Low-poly trees and rocks
- Flat colors
- No detailed textures

Purpose: establish scale, mood, and navigation.

### Level 2: Production-ish Art

- Better silhouettes
- Variant sets
- Cleaner materials
- LODs
- Optimized exports

Purpose: make the world feel cohesive.

## Learning Plan for 3D Asset Creation

### Step 1: Learn Blender Basics

Focus only on essentials:

- Navigation
- Add mesh primitives
- Edit mode
- Move/rotate/scale
- Extrude
- Bevel
- Shade flat/smooth
- Assign materials
- Set origin
- Export GLB

Avoid advanced sculpting, rigging, animation, and complex texturing at first.

### Step 2: Make One Rock

Rocks are forgiving and useful.

Process:

1. Add ico sphere or cube.
2. Deform shape.
3. Shade flat.
4. Assign gray/brown material.
5. Export GLB.
6. Place in game.

### Step 3: Make One Pine Tree

Simple approach:

1. Cylinder trunk.
2. Several cone/frustum needle layers.
3. Brown trunk material.
4. Dark green needle material.
5. Origin at trunk base.
6. Export GLB.

### Step 4: Make Variants

Do not make one perfect asset. Duplicate and adjust:

- height
- width
- branch/needle shape
- color shade
- trunk bend

### Step 5: Make the Compass

Because the compass is core to the game identity, it can be upgraded gradually:

1. Flat placeholder model.
2. Simple 3D hinged box/needle.
3. Textured/marked compass face.
4. Animated inspection state.

Source files should be committed if the repository can handle them. If assets become large, consider Git LFS later.

## Import/Export Checklist

Before an asset is considered usable:

- Correct real-world scale
- Origin/pivot is correct
- Object rests on ground at Y=0 after import
- File name is clean
- Mesh/material names are clean
- No unused cameras/lights exported unless intentional
- Looks correct in Babylon
- Performs acceptably when instanced many times
- Style matches existing assets

## Buy, Download, or Make?

### Make Yourself

Best for:

- Signature landmarks
- Compass/knife/shovel
- Core visual identity assets
- Simple stylized trees/rocks once you learn Blender basics

### Download Free/Paid Assets

Useful for:

- Temporary placeholders
- Learning examples
- Non-core props

Recommended sources:

- **Kenney** — https://kenney.nl/assets
  - Excellent for clean prototype assets and simple stylized game assets.
- **Quaternius** — https://quaternius.com
  - Free low-poly game assets, useful for early blockouts.
- **Poly Pizza** — https://poly.pizza
  - Free low-poly models, good for placeholders and simple props.
- **Sketchfab** — https://sketchfab.com
  - Huge library of free and paid assets. Quality and license terms vary.
- **itch.io Game Assets** — https://itch.io/game-assets
  - Good indie-friendly asset packs. Search for low-poly forest, survival, camping, hiking, and nature props.
- **CGTrader** — https://www.cgtrader.com
  - Large 3D marketplace. Useful, but many models are not game-ready.
- **TurboSquid** — https://www.turbosquid.com
  - Large older marketplace. Often more realistic/high-poly.
- **Fab / Unreal Marketplace** — https://www.fab.com
  - Good environment packs, but check formats and licenses before assuming they are usable in a web/Babylon project.
- **Unity Asset Store** — https://assetstore.unity.com
  - Many stylized/low-poly nature packs. Check whether the license permits use outside Unity.

Texture/material sources:

- **Poly Haven** — https://polyhaven.com
  - Free HDRIs, textures, and models. CC0.
- **ambientCG** — https://ambientcg.com
  - Free PBR materials. CC0.
- **ShareTextures** — https://www.sharetextures.com
  - Free and paid textures.
- **TextureCan** — https://www.texturecan.com
  - Free PBR textures.

Useful search terms:

- `low poly pine forest`
- `stylized forest pack`
- `low poly rocks`
- `low poly grass clumps`
- `stylized nature props`
- `low poly survival tools`
- `camping props`
- `hiking props`
- `compass 3d model`
- `shovel low poly`
- `knife low poly`

Be careful:

- Check licenses.
- Prefer CC0, clear paid royalty-free licenses, or licenses that explicitly allow commercial use.
- CC-BY can be usable, but requires attribution.
- Avoid assets marked editorial only.
- Avoid mixing mismatched art styles.
- Many store assets are too high-poly or too realistic.

For early prototyping, start with Kenney, Quaternius, Poly Pizza, and Poly Haven.

### Procedural Generation

Best for:

- Terrain
- Grass distribution
- Tree placement
- Rock placement
- Trail masks
- Color/material blending

Not a replacement for source assets. Procedural systems still need good building blocks.

## Risks

### Risk: Art production becomes too big

Mitigation:

- Keep the initial set tiny.
- Use placeholders shamelessly.
- Focus on reusable modular assets.

### Risk: Assets do not match style

Mitigation:

- Create a small color palette early.
- Compare all assets in one test scene.
- Prefer simple flat/stylized materials.

### Risk: Performance issues from vegetation

Mitigation:

- Use instancing/thin instances.
- Keep tree variants low-poly.
- Add LODs later.
- Avoid dense unique meshes.

### Risk: Scale feels wrong

Mitigation:

- Use meter-based references.
- Test assets next to player eye height.
- Walk around every asset in game before accepting it.
